(fig. 01) — Features

BibSure brings citation validation directly into your LaTeX/BibTeX workflow in VS Code. Get inline diagnostics, hover tooltips on \cite{} keys, real-time validation, and a dedicated results panel — without leaving your editor.

Inline Diagnostics Red/yellow squiggles on unverified or fake citations directly in .bib files.
Hover Tooltips Hover over @key to see CrossRef match status, DOI, and confidence score.
Validate on Save Automatically checks all citations whenever you save a .bib file.
Status Bar Live count of verified / fake / partial citations always visible in the status bar.
Results Webview Full results panel with all citations, confidence scores, and CrossRef links.
Workspace Scan Validate all .bib files in your project at once with a single command.

(fig. 04) — Architecture

Extension architecture.

vscode-extension/ ├── package.json ← Extension manifest, commands, settings ├── src/ │ └── extension.js ← Main extension logic │ ├── activate() ← Registration of commands, hover, events │ ├── validateDocument() ← Parses .bib → CrossRef → diagnostics │ ├── parseBib() ← BibTeX AST parser with Range tracking │ ├── validateOne() ← CrossRef DOI / title lookup │ ├── scoreMatch() ← Confidence scoring │ ├── provideHover() ← Hover tooltip provider │ ├── updateDecorations() ← Inline ✓/✗ icons │ └── showResultsPanel() ← Webview HTML panel
How it works
→
On file open/save, parseBib() extracts all @article{} entries with their source ranges.
→
Each citation is checked via CrossRef DOI lookup first, then title/author search as fallback.
→
Confidence is scored by title similarity + year match. Threshold defaults to 80%.
→
Diagnostics (red/yellow squiggles) are applied to the exact source range of each entry.
→
Status bar updates to show: $(check-all) BibSure: 12 OK | 2 fake
